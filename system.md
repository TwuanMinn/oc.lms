ROLES & ACCESS

There are 3 roles: Admin, Teacher, and Student
Each role has its own dashboard and can only see what is relevant to them
No self-registration — all accounts are created or imported by the Admin


ADMIN

Admin can upload a CSV file to bulk-create student and teacher accounts. The CSV should contain: name, email, role, ID, and password
Admin creates courses by entering: course name, course code, start date, assigned teacher(s), and enrolled students
Admin sets the course weekly progress (Week 1, Week 2, Week 3, etc.) with a short description for each week so students know where they are in the course
Admin can view all attendance records submitted by teachers across all courses
Admin can export attendance data as a PDF report. The report shows each student's name, ID, date of each session, and their status (Present, Absent, or Late), plus a summary count at the end
Admin can filter the attendance report by course, student, or date range before exporting


TEACHER

Teacher can see only the courses assigned to them
For each course session, the teacher manually marks every student as either: Present, Absent, or Late
Once the teacher submits attendance, it immediately reflects on the Admin dashboard
Teacher can create a test or quiz for their course and set a future date and time for it to open
On that scheduled date and time, the test automatically becomes visible to students
The test can include a title, instructions, questions, answer choices, a time limit, and a closing deadline


STUDENT

Student can see all courses they are enrolled in
Student can see the course weekly progress published by the Admin, showing which week is currently active and what it covers
Student can view their own attendance record showing each session and whether they were Present, Absent, or Late
When the scheduled test date and time arrives, the test appears on the student's dashboard and they can take it
After submitting the test, the student sees a confirmation that their answers were received


ATTENDANCE

Attendance is marked manually by the teacher per session
Each record includes the student name, session date, and status: Present, Absent, or Late
All attendance data is visible to the Admin in real time
Admin can export the full attendance report as a PDF at any time


COURSE PROGRESS

Admin defines the weekly structure of each course (Week 1, Week 2, Week 3, and so on)
Each week has a label and short description
Admin controls which week is currently active
Students can see the full week list and know exactly which week they are currently on


TESTS

Teacher creates a test inside a course and sets an open date and time
The test is hidden from students until that exact date and time
Students can only take the test within the allowed time window
After submission, the student cannot retake the test


CSV IMPORT

Admin uploads a CSV to generate multiple accounts at once
The CSV format includes: full name, email address, role (student or teacher), ID number, and default password
Once uploaded, all accounts are created instantly and users can log in


PDF EXPORT

Admin selects a course and date range, then exports a PDF attendance report
The PDF includes a table with student names, IDs, each session date, and attendance status
A summary at the bottom shows total Present, Absent, and Late counts per student
